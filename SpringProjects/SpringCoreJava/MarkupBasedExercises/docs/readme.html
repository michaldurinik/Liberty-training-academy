Spring Markup Driven Exercises

Examine the package exercise.start. This contains the flight booking system we used on previous exercises, except that now we are using a ClassPathXmlApplicationContext which reads from the file exerciseStart.xml. At the moment all that is being done in the XML file is enabling component scanning - all DI is still done via annotations.

Exercise One

Introduce XML based dependency injection for:

  1. Injecting the BookingControllerMock into the FlightBookingGUI
  2. Injecting the FlightSearchEngineMock and FlightBookingServiceMock into the BookingControllerMock

Exercise Two

Extend your solution to Exercise One by introducing XML based dependency injection for the FlightBookingGUI constructor. This should remove all DI related annotations from the code and enable you to take component scanning out of the XML file.
