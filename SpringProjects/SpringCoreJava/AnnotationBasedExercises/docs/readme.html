Spring Annotation Driven Exercises

Exercise One

Examine the package exercise.start. You will find a Program class that launches a FlightBookingGUI which enables users to search for and book flights. If you look inside FlightBookingGUI you will find that it depends on a BookingController, which in turn depends on a FlightSearchEngine and FlightBookingService. At the moment all dependencies are hardcoded.

Introduce dependency injection and the Spring framework as follows:

  1. Extract interfaces from the BookingController, FlightSearchEngine and FlightBookingService
  2. Declare the FlightBookingGUI class as a Spring service called 'gui' and the dependencies as unnamed components
  3. Inject the BookingController into the FlightBookingGUI via properties and autowiring
  4. Inject the FlightSearchEngine and FlightBookingService into the BookingController via properties and autowiring
  5. Create an AnnotationConfigApplicationContext within the Program class and configure it to use component scanning
  6. Ask the Spring context to build the bean called 'gui' and invoke 'setVisible' as before

Exercise Two

Extend your solution to Exercise One so that all components have names and are injected by name (using either @Autowired with @Qualifier or @Resource).

Exercise Three

Extend your solution to Exercise Three so that the AnnotationConfigApplicationContext object uses a configuration object which:

  1. Enables component scanning for the GUI class via @ComponentScan
  2. Contains bean provider methods to build the BookingController, FlightSearchEngine and FlightBookingService

Note that only the GUI class should now need to be explicitly declared as a component.

Exercise Four

Extend your solution to Exercise Three to include constructor injection as follows:

  1. Delete the default constructor in FlightBookingGUI
  2. Change the visibility of the remaining constructor to 'public'
  3. Mark the constructor as @Autowired and inject the arguments via @Value

Hint You can modify the methods in the Utilities class and place them within a named Spring bean. This will enable them to be called via EL expressions.
